## S22 methodology-doc IDEA tickets

- `IDEA-S22-001` - HSY `eps_diluted` source-data tag gap. The 9-ticker fixture has 9 securities but only 8 reach the headline scoreboard because HSY's `eps_diluted` is not consistently emitted in the 10-K / 10-Q XBRL filings; `next_fy_target` returns None for every HSY row. Filed as an explicit ticket so a reader of the headline does not see silent ticker dropping. Resolution path: add a fallback EPS derivation from `net_income / shares_diluted` in the S2 normalize step when `eps_diluted` is missing, and re-measure HSY coverage.
- `IDEA-S22-002` - Subprocess isolation for TiRex on Apple Silicon. The libomp interop deadlock between LightGBM and PyTorch (loaded by tirex-ts) is currently worked around by `OMP_NUM_THREADS=1` plus `torch.set_num_threads(1)`. A cleaner fix is to run TiRex inference in a child process (`multiprocessing.spawn`) or via a small RPC server, so LightGBM and torch never share an OpenMP runtime. Also covers a CUDA / MPS device path for users on hardware where the OMP serialisation is a throughput blocker.
- `IDEA-S22-003` - TiRex short-history fallback chain. SNOW (IPO 2020-Q3) has fewer than 12 quarters of EPS history through most of the 2020-2023 evaluation window, so 39 of 43 SNOW rows in the headline carry no TiRex prediction. The proprietary spec references a "short-history fallback chain" (spec line 278); v1.0 returns None and lets the cold-start equal-weight blend take over. A future fallback could use partial-year monthly or quarterly mixed-frequency context.
